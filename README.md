# pz-SOLID

## 1. Аналіз "Анти-SOLID"
У файлі `bad_code.ts` клас `Order` сам відправляв повідомлення. Це порушує **SRP** (один клас — одна задача).

## 2. Що змінено (Рефакторинг)
*   **SRP:** Відправку винесено в окремий клас `SmsNotifier`.
*   **DIP:** Клас `Order` тепер не знає, як саме шлеться повідомлення, він просто використовує інтерфейс `INotifier`.
*   **OCP:** Тепер ми можемо додати `EmailNotifier`, не змінюючи код класу `Order`.
